# رفيق الهدى — مشروع Android المستعاد

هذه النسخة محفوظة خصيصًا حتى يمكن مواصلة تطوير تطبيق **رفيق الهدى** دون الاعتماد على ملف APK فقط.

## حالة المشروع

- التطبيق: رفيق الهدى
- الإصدار الجاري: `4.8.0`
- التقنية: Android Java + WebView + HTML/CSS/JavaScript محلي
- الحزمة: `com.mastermedia.quranoffline.lite`
- فرع التطوير: `develop/rafiq-alhuda`
- فرع النسخة الاحتياطية: `backup/rafiq-alhuda-v4.7-recovered`
- طلب السحب للمراجعة: `#10`
- قارئ القرآن: 604 صور كاملة من مصحف المدينة تُجهّز أثناء البناء دون قص ودون إطار برمجي مضاف
- المحتوى: قرآن، أحاديث مختارة، أذكار، مواقيت الصلاة، القبلة، المفضلة، الملاحظات والبحث
- الصوت: تلاوة ياسر الدوسري لـ114 سورة بصيغة Ogg/Opus تعمل دون إنترنت
- التشغيل: خدمة أمامية مع MediaSession وتحكم من إشعار أندرويد
- الحجم المستهدف: قرابة 105–125 MiB مع تكييف معدل الصوت حسب مدة التلاوة الأصلية

## البناء محليًا

يتطلب المشروع Java 17 وAndroid SDK 35 وGradle 8.9. ملفات المصحف والصوت والحديث تُجهّز أولًا بواسطة Workflow أو سكربتات البناء.

```bash
cd quran-offline
gradle :app:assembleRelease --stacktrace
```

ملف APK غير الموقّع يظهر عادة هنا:

```text
quran-offline/app/build/outputs/apk/release/app-release-unsigned.apk
```

## البناء عبر GitHub Actions

الملف الرئيسي:

```text
.github/workflows/quran-apk.yml
```

هو المسؤول عن:

1. تنزيل وتجهيز صفحات المصحف الـ604 مع الحفاظ على الصورة كاملة.
2. تجهيز بيانات القرآن والحديث المختار.
3. تنزيل خط المصحف والأذان.
4. تنزيل تلاوة السور الـ114 والتحقق من كل ملف.
5. حساب معدل ضغط مناسب تلقائيًا لاستهداف نحو 93 MiB للصوت.
6. بناء APK والتحقق من وجود آخر سورة وآخر صفحة والملفات الجديدة.
7. رفع APK غير الموقّع كـ GitHub Actions Artifact.
